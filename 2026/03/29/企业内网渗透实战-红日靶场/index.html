前言

本靶机是由红日安全团队打造的免费开源网络安全实战靶场，专注于企业级内网渗透与红队攻防演练，官网地址：红日靶场下载地址

一、环境搭建

下载之后有三台虚拟机，其中Windows 7 x64需要配置两张网卡，一张使用仅主机模式，另一个使用NAT模式。剩下的主机都使用NAT模式。

网络的配置只需要在vm虚拟机中虚拟网络编辑中将NAT设置成192.168.52.0，将仅主机模式设置成192.168.1.0

虚拟机的所有密码都是hongrisec@2019

Windows7 x64:192.168.52.143
Windows server 2008:192.168.52.138
Win2K3:192.168.52.141

我使用kali作为攻击机，kali机的网卡设置为仅主机模式，与Windows7 x64的外网网卡处于同一网段下。

三、内网渗透

1、服务器上线CS

使用Cobalt Strike创建一个监听器并生成64位的exe后台程序
注意这里的Host要使用CS服务器的IP

将生成的后门程序通过中国蚁剑上传至目标站点目录

接着在蚁剑打开shell执行后台程序

可以看到CS成功上线目标服务器并可以查看到屏幕截图

2、服务器提权

右键服务器会话选择权限提升，随后在新窗口中选择监听器为最初创建的监听器, 提权方式选择默认的svc-exe, 点击运行即可（或者直接在 beacon 控制台输入 elevate svc-exe 监听器名 命令，也可实现新会话提权至 system 用户；getsystem 命令可在当前会话提权至 system 用户）

执行完成之后可以看到服务器多了个会话条目，用户为 system, 提权成功

3、获取密码凭证

在 beacon 控制台中执行 logonpasswords 命令，随后在” 视图” 菜单中打开” 密码凭证” 子菜单，查看服务器的所有用户的账户密码

从上图可以看到 administrator 用户和登录密码, 所属域为 god, 密码来源为 mimikatz 工具

继续在 CS 中使用服务器自带的 nmap 扫描内网有哪些活跃主机：shell nmap -sn -PR -n 192.168.52.143/24

从上图可看出服务器内网还有 192.168.52.138 和 192.168.52.141 两台主机

7、横向移动

创建一个 SMB 监听器，名字为 2222，Payload 为 Beacon SMB

进入” 目标列表”, 右键选择 192.168.52.138 主机 (域控制器)，进入” 横向移动” 的”psexec” 配置菜单，在菜单中选择监听器为前面创建好的 2222，会话选择 Admin 用户，将下面的选项打勾，然后点击运行

最后一台域成员主机 192.168.52.141 也如法炮制，成功以 System 用户身份上线 CS

至此已经获得了所有主机的最高权限

四、总结

这是一个非常完善的企业级靶场，从外网打点渗透到某台主机并以此为跳板进行提权和横向移动，整个攻击链下来非常的完整和清晰，是一个非常值得训练的靶场。在渗透过程中也遇到过问题，就比如在横向移动的时候，遇到无法jump的情况，经过检查发现是用作跳板机所用的权限不够导致无法连接，还有就是要时刻关注主机上线的情况，通过last的数值判断主机是否在线。
